## Integration Findings

### Critical Pre-Wiring Risks (fix before Phase 3)

**INT-01: `categories` field shape** (`api/src/main.py:191`)

`/predict` returns:
```json
"categories": [{"cardboard": 0.92}, {"paper": 0.05}]
```
Should return:
```json
"categories": [{"label": "cardboard", "confidence": 0.92}, ...]
```
Affects: CLSF-01, CLSF-02. Phase 3 must extract labels from alternatives — dynamic-key dicts require `Object.keys()[0]` which is fragile.

**INT-02: No council slug format contract** (`api/src/guidelines.py:120-135`)

`GET /advice` accepts `council_slug` as a free-form string. The required format is `{CouncilNameState}` PascalCase (e.g. `SydneyNSW`, `MelbourneVIC`). Nominatim returns plain strings (`"Sydney"`, `"New South Wales"`). A malformed slug reaches `_rny_url()`, produces an invalid URL, and the LLM may answer with `is_fallback=False` from training data — silent wrong advice.

**INT-03: Confidence threshold undefined**

The 0.65 threshold from REQUIREMENTS.md does not exist as a config constant. Phase 3 must introduce `CONFIDENCE_THRESHOLD` as a `Settings` field to avoid hardcoding it.

### Significant (address before Phase 4)

**INT-04: `/predict` vs `/upload` input format mismatch**

- `POST /predict`: `multipart/form-data` (`UploadFile`)
- `POST /upload`: JSON body (`image_base64: str`)

Phase 4 mobile UI captures images for both flows and must transcode between formats. WEBP images pass `/predict` content-type check but fail `/upload` magic byte check (`_is_valid_image()` only handles JPEG and PNG).
